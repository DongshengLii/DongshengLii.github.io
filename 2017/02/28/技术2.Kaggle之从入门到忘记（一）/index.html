Kaggle之从入门到忘记（一）

发表于 2017-02-28 | 分类于 一本正经谈技术 | | 阅读次数
序言和Python环境准备

序言

在数据挖掘的课上，王老师给大家介绍了一个数据挖掘竞赛平台——Kaggle：此平台不仅提供比较有价值的数据和需求，而且做得好的话还可以获得奖金。对于计算机这门以项目驱动学习的学科而言，拿它练手实在是再好不过！

本系列主要是建立在《机器学习与实践》一书（后文简称为“书”）的基础上的，结合一点针对自己不足的查漏补缺。之所以起这么一个搞怪的名字是因为数据挖掘与其说是一门学科一个专业，更不如说是一门技术，而技术真正掌握的标志就是化有形为无形，真正融入到自己的思维意识之中，所以“忘记”在我看来就是最高的境界啦~希望我在学习的同时也能帮助到的对数据挖掘感兴趣的你！

python环境准备

数据挖掘主要使用两种语言：R或者Python。由于建模校赛使用过一次R感觉不太适合我，所以我就使用Python来完成主要任务~

Python的安装和pip的准备自然不必多说，网上有很多教程，很详细。我主要介绍一下需要用到的第三方包：NumPy&Scipy,Matplotlib,Scikit-learn,Pandas,Anaconda.这些包的下载主要还是在命令行运行如下代码： 

pip install PackageName

保持联网状态，就可以实现一键懒人快速安装啦！

在此过程中我遇到了Scipy安装失败的问题，解决方法是：在指定网站自行下载相关文件，然后在下载文件所在文件夹命令行使用如下指令安装：

pip install filename.whl

python2.X转换为3.X

本文的最后我们还需要知道，本书使用Python2编写的，而一般来讲，大家都是使用Python3.X呢对不对，所以当遇见Python2代码编译出现语法错误的时候一定要记得使用Python3自带的转换脚本“2to3.py”（位置在PYTHON\Tools\scripts）然后在需要转换的代码文件所在文件夹，命令行输入：

python 2to3.py -w filename.py

就可以转换成python3畅快地玩耍啦！（说的可能不是很详细，详情见CodeMing’s Blog）
